# Installation

Surge is available on multiple platforms. Choose the method that works best for you.

## System Requirements

Before installing Surge, ensure your system meets these requirements:

- **Operating System**: Linux, macOS, or Windows
- **Go Version**: 1.24.4+ (only if installing via `go install`)
- **Terminal**: ANSI color support recommended for the best TUI experience

## Installation Methods

<Tabs>
  <Tab title="Prebuilt Binary">
    ### Download Prebuilt Binary

    The easiest method—just download and run.

    <Steps>
      <Step title="Download the Latest Release">
        Visit the [Surge Releases page](https://github.com/surge-downloader/surge/releases/latest) and download the binary for your platform.
      </Step>
      
      <Step title="Extract the Archive">
        Extract the downloaded archive to a directory of your choice.
      </Step>
      
      <Step title="Make it Executable (Linux/macOS)">
        ```bash
        chmod +x surge
        ```
      </Step>
      
      <Step title="Move to PATH (Optional)">
        For easy access from anywhere:
        
        **Linux/macOS:**
        ```bash
        sudo mv surge /usr/local/bin/
        ```
        
        **Windows:**
        Add the surge directory to your PATH environment variable.
      </Step>
      
      <Step title="Verify Installation">
        ```bash
        surge --version
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="macOS (Homebrew)">
    ### Install via Homebrew

    Recommended for macOS and Linux users with Homebrew installed.

    <Steps>
      <Step title="Add the Surge Tap">
        ```bash
        brew tap surge-downloader/tap
        ```
      </Step>
      
      <Step title="Install Surge">
        ```bash
        brew install surge-downloader/tap/surge
        ```
      </Step>
      
      <Step title="Verify Installation">
        ```bash
        surge --version
        ```
      </Step>
    </Steps>

    <Info>
    Homebrew will automatically add Surge to your PATH and handle updates via `brew upgrade surge`.
    </Info>
  </Tab>

  <Tab title="Windows (Winget)">
    ### Install via Winget

    Recommended for Windows users.

    <Steps>
      <Step title="Install Surge">
        Open PowerShell or Command Prompt and run:
        ```bash
        winget install surge-downloader.surge
        ```
      </Step>
      
      <Step title="Verify Installation">
        ```bash
        surge --version
        ```
      </Step>
    </Steps>

    <Tip>
    Winget will automatically handle PATH configuration and future updates.
    </Tip>
  </Tab>

  <Tab title="Arch Linux (AUR)">
    ### Install via AUR

    For Arch Linux users with an AUR helper like `yay`.

    <Steps>
      <Step title="Install Surge">
        ```bash
        yay -S surge
        ```
      </Step>
      
      <Step title="Verify Installation">
        ```bash
        surge --version
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Go Install">
    ### Install via Go

    Requires Go 1.24.4 or later.

    <Steps>
      <Step title="Verify Go Installation">
        ```bash
        go version
        ```
        
        Ensure you have Go 1.24.4 or later installed.
      </Step>
      
      <Step title="Install Surge">
        ```bash
        go install github.com/surge-downloader/surge@latest
        ```
      </Step>
      
      <Step title="Verify Installation">
        ```bash
        surge --version
        ```
      </Step>
    </Steps>

    <Warning>
    Ensure your `$GOPATH/bin` directory is in your PATH. You can check with `echo $GOPATH/bin` and add it to your shell profile if needed.
    </Warning>
  </Tab>

  <Tab title="Docker">
    ### Run via Docker Compose

    Run Surge in server mode with Docker Compose.

    <Steps>
      <Step title="Download the Compose File">
        ```bash
        wget https://raw.githubusercontent.com/surge-downloader/surge/refs/heads/main/docker/compose.yml
        ```
      </Step>
      
      <Step title="Start the Container">
        ```bash
        docker compose up -d
        ```
      </Step>
      
      <Step title="Get the API Token">
        ```bash
        docker compose exec surge surge token
        ```
        
        Save this token—you'll need it to authenticate API requests and connect remotely.
      </Step>
      
      <Step title="Verify Installation">
        Check downloads and API availability:
        ```bash
        docker compose exec surge surge ls
        ```
      </Step>
    </Steps>

    <Info>
    View logs with:
    ```bash
    docker compose logs -f surge
    ```
    </Info>
  </Tab>
</Tabs>

## Post-Installation

### Generate API Token

Surge uses a bearer token for API authentication. Generate or view your token:

```bash
surge token
```

<Note>
The token is automatically generated on first run and stored in `~/.local/state/surge/token` (Linux) or equivalent on other platforms.
</Note>

### Configuration Directories

Surge uses the following directories:

- **State**: `~/.local/state/surge/` (database, token)
- **Logs**: `~/.local/state/surge/logs/`
- **Settings**: `~/.config/surge/settings.toml`
